Basic HTML and HTML5

HTML uses a special syntax to describe the structure of a web page to a browser.

   <!-- An example of a simple html doc -->

     <h1>Hello World</h1>

</html>

Basic CSS

Cascading Style Sheets tell the browser how to display the content written in HTML. Styling options include color, font, size, spacing, etc.

Selectors are used to indicate which CSS rules apply to which HTML elements. Below are some examples of common, useful selectors (selecting by element, by .class, by #id, and by parent_type child_element. When using parent_type child_element selector, the first argument can be element, .class, or #id, while the second argument must be an element type.

h1 {
  font-size: 24px;
}

.class-name {
  text-transform: uppercase;
}

#id-name {
  text-indent: 30px;
}

p h1{
  font-size: 20px;
}

The most dynamic tool that I learned in this section is the media query. Using media query, HTML variables' styles can be dependant on the conditions of the viewport's width or height. The example below details css rules that apply when the width of the viewport is less than 350px.

@media (max-width: 350px) {
  p {
    /* css rules */
  }
}

Applied Accessibility

Accessibility refers to a UI that can be understood, navigated, and interacted with by a broad audience, including those with visual, auditory, mobility, or cognitive disabilities.

Some practices that improve accessiblity:

  • image & audio have a text alternative
  • heading level reflects hierarchical relationships
  • div types are most descriptive
  • group content & caption into a figure
  • 'label for' your form fields
  • fieldset for your radio buttons
  • use sufficient contrast
  • avoid colorblindness issues
  • utilize tabIndex for keyboard focus
  • give links meaningful link text

Responsive Web Design Principles

There are many devices that can access the web, and they come in all shapes and sizes. Responsive web design is the practice of designing flexible websites that can respond to different screen sizes, orientations, and resolutions.

Making images responsively scale is straightforward!

img {
  max-width: 100%;
  height: auto;
}

Scale down the height & width of a low-res img so that it renders well on hi-res screens.

Use viewport units to make typography responsize:

  • vw (viewport width)
  • vh (viewport height)
  • vmin (viewport small axis)
  • vmax (viewport large axis)

CSS Flexbox

Flexbox is a powerful, well-supported layout method that allows for ease in centering elements on the page and creating dynamic user interfaces that shrink and expand automatically.

Placing the CSS property 'display: flex;' on an element allows you to use other flex properties to build a responsive page.

#container {
  display: flex;
}

One of these properties is 'flex-direction', which can have a value of 'row', 'column', 'row-reverse', or 'column-reverse'.

#container {
  flex-direction: row;
}

I learned to justify or align both individual and groups of items along the main & cross axes.

#container {
  align-items: center;
}

The flex-wrap property tells CSS to wrap items into multiple rows or columns. There are multiple different values for this property:

  • nowrap: default setting, all items displayed on singular row or column
  • wrap: wraps columns left to right and rows top to bottom
  • wrap-reverse: wraps columns right to left and rows bottom to top

The flex-shrink and flex-grow properties allow items to shrink/grow when a flex container is too small/large (the number taken as an argument specifies relative shrink/grow compared to other items). The flex-basis property specifies an initial size.

#shrinks-more {
  flex-shrink: 2;
}

#shrinks-less {
  flex-shrink: 1;
}

Items in a container can be assigned an 'order' property, which informs CSS how to arrange the items in the container.

#second-item {
  order: 2;
}

#first-item {
  order: 1;
}

CSS Grid

The CSS grid is a newer standard that makes it easy to build complex responsive layouts. It works by turning an HTML element into a grid, and lets you place child elements anywhere within. An element can be turned into a grid by setting its display property to grid.

#container {
  display: grid;
}

To define the structure of the grid, grid-template-rows or grid-template-columns is used. The arguments can be (mixed) in relative or absolute units. The following container has 5 columns. The first is as wide as its content, the second is 30px, the third is 20% of its container, and the fourth and fifth are 3/4 and 1/4 of the remaining space, respectively. Even the minmax() function can be used as this property value.

#container {
  grid-template-columns: auto 30px 20% 3fr 1fr;
}

Items can span multiple rows or columns by setting their grid-row or grid-column properties. The following snipet shows an item spanning columns 2-4, using the column lines 2-5 as arguments.

#wide-item {
  grid-column: 2 / 5;
}

Similarly to flex displays, grid items can be justified or aligned, both in groups and individually, along the main and cross axes. The default value is stretch, but it accepts other value also:

  • start
  • center
  • end

Unlike flex displays, grids allow for items for span both rows and columns. A grid-template-areas can be defined, and later items assigned to areas.

#container {
  grid-template-areas:
    "header header"
    "advert content"
    "advert content";
}

#advert-item {
  grid-area: advert;
}

The repeat function can be used to specify the number of times a row or column should be repeated. This can be combined with the auto-fill and auto-fit functions, which automatically instert as many rows or columns as possible depending on the size of the container & size of the items.
